vpp asan 相关原理介绍

通过 mmap 映射的内存，

clib_mem_vm_map_internal().clib_mem_unpoison()
将 clib_mem_vm_map 前面的 clib_mem_vm_map_hdr_t 和 此次 mmap 的 clib_mem_vm_map 进行解毒，
以便后面可以访问，
clib_mem_vm_map_internal() 主要是 mmap 一段将要使用的 内存，
调用该接口后可以直接读写内存(不是 PROT_NONE 状态)，
vpp 很多地方分配内存都使用了该接口，

注: dlmalloc 管理的 heap 都是通过 clib_mem_vm_map_internal() 分配的，但 clib_mem_vm_map_internal() 分配的内存不一定用于 heap ，如 vpp thread stack 、 vpp process stack 或 vpp memif 使用该接口分配的内存，

asan 与 dlmalloc 相关接口

总结

创建 mheap 时会将几乎所有区域 poison ，
分配时会将对应内存 unpoison ， 释放时会将对应内存 poison ，
注: vpp 没有像 dpdk 那样在 分配区间的 header 和 trailer 处 poison ，应该检查不出微小的越界访问，

创建 mheap

clib_mem_create_heap_internal().clib_mem_poison()
clib_mem_create_heap_internal(): 通过 dlmalloc 创建 mem heap ，
这里主要是从 ms->least_addr 开始 poison ， poison 的长度为 ms->footprint ，
ms->least_addr 在这个函数里为: mmap 映射内存的 起始地址 + strlen(name) ，
ms->footprint 在这个函数里为: mmap 映射内存的 size - strlen(name) ，

分配，

clib_mem_heap_alloc_inline().clib_mem_unpoison()
clib_mem_heap_alloc_inline() 主要是从 dlmalloc 管理的 mem heap 内分配内存，
内存分配出来后，需要进行解毒，这样后面才可以使用，

释放

clib_mem_free_s().clib_mem_unpoison()
clib_mem_free_s() 主要是从 dlmalloc 管理的 mem heap 内释放内存，
clib_mem_free_s().clib_mem_free().clib_mem_heap_free().mspace_free()
这个接口使用较少，
这里的解毒主要是为了将数据 清0 ，

__clib_export void
clib_mem_free_s (void *p)
{ 
	uword size = clib_mem_size (p);
	clib_mem_unpoison (p, size);
	memset_s_inline (p, size, 0, size);
	clib_mem_free (p);
}

通常的释放不会经过上面的函数 clib_mem_free_s() ，
clib_mem_free() 或 (_vec_free() 也会调用下面的接口)
.clib_mem_heap_free().clib_mem_poison()

asan 与 vec 相关接口，

vec 内存属于 vpp mheap(通过 dlmalloc 管理) ，也就是 通过 dlmalloc 接口进行分配和释放，
分配时与 dlmalloc 类似，就是将多余的空间 poison ，
不一样的地方主要是 vec 通过 dlmalloc 分配出来的内存有时不一定都使用，需要将后面不使用的内存 poison ，
缩小 vector 长度时， 因为 不会释放内存到 dlmalloc heap ，
需要通过 _vec_update_len() 或 _vec_set_len() 将后面已经分配出来但没有使用的区域进行 poison ，
增加 vector 长度， 但没有超过 vector 已经分配的长度时， 此时不会再从 dlmalloc heap 中分配，
需要通过 _vec_update_len() 或 _vec_set_len() 将需要的 额外内存 unpoison ，

分配，

_vec_alloc_internal().clib_mem_poison()
_vec_alloc_internal(): vpp vector 涉及到内存分配的很多流程都会使用这个接口，
如 vec_validate().vec_validate_hap()._vec_validate()._vec_alloc_internal()
_vec_add()._vec_alloc_internal()
_vec_add2()._vec_alloc_internal()
_clib_fifo_resize()._vec_alloc_internal()
_pool_init_fixed()._vec_alloc_internal()
通过 clib_mem_heap_alloc_aligned().clib_mem_heap_alloc_inline() 从 mheap 里分配内存，
分配接口分配后会对相应内存区域解毒，(dlmalloc heap 创建或释放 时都会 进行 poison)
随后会将多分配的但用不到的内存给 poison ，
clib_mem_poison (p + req_size, alloc_size - req_size);

调整 vector 长度，

_vec_resize_internal()._vec_realloc_internal()

增加长度时 和 分配类似， 会再次从 dlmalloc 里进行分配，
减少长度时 因为 不会释放内存到 dlmalloc heap ，这样 dlmalloc 释放接口不会调用，
需要通过 _vec_update_len() 将后面已经分配出来但没有使用的区域进行 poison ，

还有很多其他接口，
如 _vec_resize() 等，

asan 与 pool 相关接口，

pool 的内存基于 vector ，间接基于 dlmalloc ，
但 pool 分配出来的内存并不是用户都能使用的， 只有 pool_get() 获取的内存才可以使用， pool_put() 时需要再次 poison ，

分配

pool_alloc_aligned_heap()._pool_alloc()
通过 vector 接口获取内存，
刚分配出来后因为没有给用户，需要全部 poison ，

pool_get()

_pool_get().clib_mem_unpoison()

主要将从 pool 分配出来的 element unpoison ，

pool_put()

pool_put().pool_put_index()._pool_put_index().clib_mem_poison()
主要将 返回给 pool 的 element poison ，
